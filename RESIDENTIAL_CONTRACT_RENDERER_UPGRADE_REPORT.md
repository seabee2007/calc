## Renderer Created

### `ResidentialContractDocument.tsx`

Accepts assembled sections plus display context inputs:
- `sections` — engine output (unchanged content)
- `documentTitle`, `answers`, `selectedProject`, `companySettings`
- `disclaimer`, `risk`, `complianceIssues`

Layout mirrors Change Order quality:
- **Header** — company logo/name/address/contact/license + **RESIDENTIAL CONTRACT / Draft / Date** box
- **Project & parties card** — contractor, owner, property, project type
- **Summary boxes** — contract summary, legal/draft notice, risk summary
- **Sections** — uppercase labels, semibold titles, relaxed body text, list support
- **Signature blocks** — contractor and owner/client bordered cards

Root uses `bg-white text-slate-950` with no dark-mode inheritance inside the paper.

## Preview Router Changes

`DocumentPreviewRouter` architecture:

```
change_order          → ChangeOrderDocument (paper shell)
residential_contract  → ResidentialContractDocument (paper shell)
previewVersion set    → PreviewPanel (historical snapshot)
other types           → PreviewPanel (generic)
```

Extracted `PaperPreviewShell` wrapper (Paper preview label + optional Internal badge) used by both Change Order and Residential Contract live previews.
